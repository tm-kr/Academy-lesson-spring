주제 : 여행 다이어리
- 다녀온 여행에 대한 기록을 일기처럼 포스팅하는 웹 사이트이다.
- 지역별 여행지에 대한 정보를 공유할 수 있다.

구현해야될 기능
- 회원가입 : 어떤 정보를 입력받을지, 본인인증 관련
- 로그인 : 구글/네이버 등 로그인할 수 있는 폼구현(회원가입도 맞춰서 구현)
- 회원 정보 수정 및 탈퇴 : 탈퇴 시 작성한 게시글에 대한 처리
- 엠블럼 : 무엇을 기준으로 엠블럼을 만들 것인지
- 게시글 작성 : 게시글 작성 시 필요한 요소등
- 게시판 : 게시글이 어떤 형식으로 보여질지
- 메인페이지, 각 페이지 : 각 페이지마다 어떤것을 보여줄 것인지 url은 어떤식으로 이루어져야 하는지, 다크모드관련
- 프로필(마이페이지) : 아래에 있는 내용 구현
 1. 본인 사진
 2. 배경 사진
 3. 간단한 소개(상태메세지)
 4. 다녀온 여행지를 지도에 기록
 5. 본인이 작성한 게시글에 대한 통계자료
 6. 본인이 작성한 게시글을 확인
 !!!!! 프로필 : 마이페이지에 대해 어떤식으로 구현할 것인지 정확하게 나누기
- 검색기능 : 지역별, 날짜별 검색등 검색에 필요한 정보가 무엇인지 어떤식으로 검색기능을 만들것인지
- 열람했던 게시글을 다시 확인 : 추천등과 같이 특정 상황에서 글을 확인할 것인지, 로그기록을 통해 확인할 것인지 등
- 추천, 비추천
- 댓글
- 채팅
+ 관리자 페이지 : 신고등 가입한 회원에 관한 정보를 관리할 수 있는 페이지

---------------------------------------------------

공통기능
1.게시판
 - 게시판 종류 및 보여주는 방식은 자유롭게 가져오시면 됩니다.
2.회원가입
 - 회원가입 시 필요한 정보(닉네임 등)는 하고 싶은 주제에 맞춰 생각해오시면 됩니다.
3.회원관리
 - 정보수정, 탈퇴 : 탈퇴 시 등록된 회원정보 삭제, 아이디/비밀번호 찾기 등
4.엠블럼
 - 출석, 게시글 추천 등 특정 이벤트가 진행됐을 때 사용 : 당근마켓 참고
5.검색기능
 - 게시글 제목, 사용자 아이디, 사용자 닉네임등 검색에 필요한 정보도 주제에 맞게 생각해보시면 됩니다.
6.로그인
 - 아이디 기억하기 등 필요한 기능있으면 추가로 생각해오시면 됩니다.
7.회원탈퇴 시 해당 회원에 관련된 데이터 처리 관련
 - 탈퇴회원에 대한 정보를 어디까지 삭제할 것인지 : 작성한 게시글 모두 삭제, 개인정보 삭제 등 가볍게 생각해보시면 될 것 같습니다.
